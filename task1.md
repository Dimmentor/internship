# Этап 1. Мультиагентный граф

Создайте мультиагентную систему на основе графа.

## Требования

* Используйте граф агентов (LangGraph или аналогичный инструмент).
* В графе должно быть несколько взаимодействующих узлов.
* Каждый агент должен иметь собственную роль, ответственность и системный промпт.
* Агенты должны обмениваться сообщениями между собой для решения пользовательской задачи.
* В системе должна присутствовать маршрутизация между агентами. Решение о том, какой агент будет выполнять следующий шаг, должно приниматься динамически.
* К проекту должна прилагаться визуализация графа в mermaid-коде/png (к примеру, у LangGraph есть встроенный инструмент draw_mermaid).

Количество агентов, структура графа и сценарий работы не ограничиваются.

Это может быть:

* линейный пайплайн (цепочка агентов);
* роутер с несколькими специализированными агентами;
* иерархия Супервайзер(выбор исполнителя) -> Исполнитель;
* граф с повторными проходами(например, код-ревью);
* любые другие архитектуры, которые вы считаете подходящими.


### Примеры

* команда разработчиков (аналитик, разработчик, ревьюер/тестировщик);
* служба технической поддержки(1 линия, 2 линия);
* документирование кода;
* собственные идеи приветствуются.
