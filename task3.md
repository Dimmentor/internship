# Этап 3. Документация

Подготовьте небольшое описание своего проекта в формате .adoc

Для этого нужно будет самостоятельно разобраться в синтаксисе .adoc, который относительно близок к .md формату, но чуть сложнее. Соответственно, .adoc имеет ряд преимуществ перед .md. 
Предварительно установить нужные плагины на IDE.

При написании документации ответьте себе на несколько вопросов, которые затем легко будет разделить на разделы документации:

* какую задачу решает система - Описание проекта;
* почему была выбрана именно такая архитектура графа - Архитектура;
* какие роли выполняют агенты - Описание узлов;
* какие инструменты используются - Описание инструментов - при наличии отдельного репозитория MCP-сервера - стоит написать отдельную документацию под этот проект;
* какие ограничения есть у текущей реализации;
* что бы вы сделали дальше при наличии дополнительного времени. (TODO)

## Реализованы ли дополнительные возможности:

* память между запросами;
* историю сообщений;
* несколько инструментов для одного агента;
* возможность выбора модели;
* логирование/трассировка работы графа;
* повторные попытки вызова инструментов;
* использование нескольких MCP-серверов;
* собственные интересные идеи.


На текущий момент, кто-то уже мог реализовать часть этого функционала - это стоит отметить в документации, и описать реализацию.
